# Dynamic Dual Grid 策略逻辑数学化重构

> 目的：把三大事件（利润搬运 / 仓位恢复 / 亏损腿减仓）的口头规则重述为一套自洽的数学模型，
> 修复 PROJECT_PROGRESS §21 记录的逻辑缺口，并给出可验证的参数一致性约束与验收标准。
> 本文是策略设计层文档；工程落点见 `design/ARCHITECTURE.md` §4.2（EventRule 流水线）与 §4.3（RiskGuard 链）。

## 1. 第一性原理：双腿只是记账，净敞口才是仓位

同一合约上同时持有多头 L 和空头 S（Hedge Mode 双腿），经济上等价于一个净头寸：

```
Δ = L − S        （单位：币数量）
```

任意区间的盈亏恒等式（无交易时段）：`dPnL = Δ · dP`。计入操作与成本后：

```
PnL = ∫ Δ(t)·dP(t)  −  Σ(手续费 + 滑点)  −  ∫ φ(t)·Δ(t)·P(t) dt
```

φ 为 funding rate（正值时多头付空头；双腿账户净支付 φ·Δ·P）。

三个直接推论：

1. **L = S 的"完美对冲"对价格完全免疫，也完全不产生收益**，只消耗手续费与保证金占用（gross = L+S 占保证金）。唯一优点：funding 净额为零——**funding 敞口 ∝ |Δ|**。
2. **腿级"已实现利润"在 Δ 不变时是记账幻觉**：盈利腿变现多少，亏损腿浮亏等额加深，净值不变，还付两笔手续费。
3. **策略赚钱当且仅当 Δ 的路径与后续价格变动正相关**（低位净多、高位净空），且该相关性收益覆盖 churn 成本与 funding。

因此三大事件必须用 Δ 重新表述：

| 事件 | 表面语义 | Δ 语义（真实经济含义） |
|---|---|---|
| 利润搬运 | 盈利腿止盈 + 补亏损腿 | **建立逆势偏斜**：涨后 Δ 变负 / 跌后 Δ 变正——一笔押注回归的方向性仓位 |
| 仓位恢复 | 回调后恢复仓位结构 | **平掉偏斜、落袋回归收益**：Δ → 0，在更优价格重新对冲 |
| 亏损腿减仓 | 趋势确认后砍亏损腿 | **先止损逆势偏斜，再建立顺势敞口**：Δ 翻向趋势方向 |

"用利润恢复亏损腿"不是资金划转，而是 Δ 的一次逆势移动；收益来自之后的回归，不来自"搬运"动作本身。若价格继续单边，这笔操作是亏的——这正是事件三（趋势止损）存在的原因。这也澄清了 §21.6 的口径问题。

## 2. 收益结构（诚实版）与品种准入公式

该策略 = **区间市收割回归 + 趋势市支付有界止损**。不存在让它在所有市况都赚钱的参数；参数的职责是让"区间收益 × 频率 > 趋势止损 × 频率 + 总成本"。

单次偏斜押注的期望（一阶近似）：在偏离 a 处建立偏斜 δ，回归则赚 δ·P·a；延伸到趋势阈值 θ 则止损 δ·P·(θ−a)；来回成本率 c = 2(f+s)（taker 费率 f + 滑点 s，建仓与平仓各一程）：

```
E = π·δPa − (1−π)·δP(θ−a) − δPc > 0    ⇔    π > 1 − (a − c)/θ
```

π 为"偏离 a 的行情在触及 θ 前回归"的概率。代入当前默认参数（a=1.5%，θ=4%，f=0.05%，s=0.02% ⇒ c≈0.14%）：**π > 66%**。

这给出可测量的**品种准入标准**：用历史数据统计每个 symbol 的 π̂ 及置信区间，π̂ 显著高于下界才对该 symbol 启用策略（§9 验收含此项）。

## 3. 统一内核：目标净敞口函数 Δ*

三条独立规则容易漂移（现状：engine.py 与 planning.py 逻辑双写、部分参数未接线）。内核统一为**一个目标偏斜函数 + 迟滞**；事件退化为"把 Δ 推向 Δ* 的具名迁移"，保留事件作为审计与展示粒度：

设 m = P/P_b − 1（P_b 为当前锚点；必要时用波动率归一化 z = m/σ̂，见 §5）：

```
区间带 |m| < θ_t：  Δ*(m) = −sign(m) · δ_unit · min( floor(|m|/a_pt), N )    # 逆势、阶梯、限 N 档
趋势带 |m| ≥ θ_t：  Δ*(m) = +sign(m) · (q_b − q_floor)                       # 亏损腿砍至地板后的顺势敞口
重锚完成后：        Δ* = 0                                                    # 新锚点上重建平衡对冲
```

- 每档变化必须过迟滞与冷却（§5），并受自融资预算约束（§6 C7）。
- **`plan_only` 与未来 `paper` / `live` 共用同一内核**：规划 = 用真实持仓算出当前 Δ 与 Δ* 的差，生成把 Δ 推向 Δ* 的动作清单。逻辑双写就此消除。

## 4. 趋势生命周期状态机（修复 §21.1 / §21.2）

"趋势"必须有生灭：确认 → 处置 → 结束判定 → 重锚 → 重建。极值与计数器的作用域是**两次重锚之间**——从根上修复"symbol 永久锁死减仓态、max_times_per_trend 退化为终身上限"。

状态：

| 状态 | 含义 | Δ 特征 |
|---|---|---|
| BALANCED | 平衡对冲 | Δ ≈ 0 |
| SKEWED_LONG / SKEWED_SHORT | 区间内逆势偏斜 | sign(Δ) = −sign(m) |
| TREND_UP / TREND_DOWN | 趋势确认、处置中 | sign(Δ) = sign(m)（地板化后） |
| REANCHORING | 趋势结束，重锚 + 分批重建 | Δ → 0 |
| PAUSED / STOPPED | 风控暂停 / 终止（吸收态） | 只减不加 / 全平 |

迁移表（触发 → 动作 → 复位语义）：

| 迁移 | 触发 | 动作 | 复位 |
|---|---|---|---|
| BALANCED → SKEWED | \|m\| ≥ a_pt，且 C1/C2/C7 通过 | 按 Δ* 建一档逆势偏斜 | — |
| SKEWED → BALANCED | 回撤自偏斜期极值 ≥ p_r，持续 T_r tick | 平偏斜（原"仓位恢复"），实现回归收益 | 档位清零 |
| SKEWED / BALANCED → TREND | \|m\| ≥ θ_t 且持续 T_t tick（§5 持续性/斜率确认） | ① 先平逆势偏斜（止损）② 分步砍亏损腿至 q_floor（步距 s、比例 ρ、冷却） | 开始记录趋势极值 |
| TREND → REANCHORING | 回撤自趋势极值 ≥ p_t，且 \|m\| 回到 θ_out 带内持续 T_e tick（迟滞 θ_out < θ_t） | 冻结加减仓 | — |
| REANCHORING → BALANCED | 分批重建完成 | P_b ← 趋势结束时现价（备选：最近摆动高/低点，§5.1 回测对比项）；两腿分 k 批重建至 q_b' = base_notional/P_b'，批间要求价格停留带内，离带则中止重评 | **high/low_since_base 与全部 \*_count_in_trend 清零** |
| 任意 → PAUSED / STOPPED | §7 不变量击穿 | 只减仓 / 拆对冲全平 | 人工复核后恢复 |

要点：
- **亏损腿重建不追旧锚**：只在 REANCHORING 按新锚点分批进行（修复 §21.2），避免趋势未死时逆势接刀。
- 趋势结束 = 回撤 + 回带 + 持续时间三条件 + 迟滞，抑制 θ_t 附近反复横跳（whipsaw 成本见 §6）。

## 5. 触发的统计标准化（修复 §21.5）

固定百分比阈值无法区分"一小时 4%"与"一个月阴跌 4%"，也不随品种波动率伸缩。标准化方案：

- **波动率估计**：σ̂ = EWMA 已实现波动（tick 收益，窗口 W），换算到判定视界 H：σ̂_H = σ̂·√H。
- **阈值随波动伸缩**：a_pt = max(k₁·σ̂_H, a_floor)，θ_t = k₂·σ̂_H，k₂/k₁ ≈ 2.5–3；a_floor 由成本下界 C1 给出。
- **持续性确认**：条件需连续 T tick 成立（去抖），替代单点触发。
- **斜率维度**（可选增强）：对 m 做窗口回归，要求斜率显著非零且方向一致——区分暴跌与叠加陈旧锚点的阴跌。
- **迟滞**：进入 θ_t、退出 θ_out ≈ (0.5–0.7)·θ_t，两阈值之间为"既成趋势维持带"。

### 5.1 已定设计决策（2026-07-08 评审确认）

1. **阈值单位 = σ，弃用固定百分比**。依据：偏斜押注的本质是押"统计上异常的偏离会回归"，偏离是否异常只能相对该币自身噪声尺度（同样 1.5%，平静期 BTC 是 3σ 信号、高波动币是 0.8σ 噪声）。公式：
   `a_pt = clamp(k₁·σ̂_H, 5c, 上限)`，`θ_t = k₂·σ̂_H`，**k₂/k₁ 比例固定**（风报几何不随波动率漂移）。
2. **k₁/k₂ 数值不由人拍**：由回测几何扫描标定（k₁ 初始扫描区间 1.5–2.5σ），人只定单位与扫描范围。
3. **K 线结构不做主触发**（可审计性：触发理由必须能在风控中心解释；可测性：形态定义主观）。收窄为两个**可计算**用途，作为回测对比开关，有增益则留、无则砍，不进核心闭环：
   - **结构化锚点**：重锚位取最近摆动高/低点（swing point），与基线"趋势结束时现价"对比；
   - **入场调制器**：偏离达标但价格距近期关键位 < 0.5×ATR 时推迟一档建偏斜。
4. **趋势生命周期三决策**（闭环评审确认）：
   - 趋势结束判定 = 三条件：回撤自极值 ≥ p_t **且** |m| 回到 θ_out 带内 **且** 持续 T_e tick；
   - 重锚基线 = 趋势结束时现价（摆动点锚为回测对比项，见上）；
   - 双腿重建 = 分 3–4 批，批间要求价格停留带内，离带中止重评。

## 6. 成本与资金约束：参数一致性不等式（修复 §21.6–§21.8）

记 c = 2(f+s) 为一来一回成本率，D = 单币种最大回撤额度，φ̄ = funding 幅度期望。

| # | 约束 | 含义 |
|---|---|---|
| C1 | a_pt ≥ 5c | 触发幅度至少 5 倍来回成本，压制手续费 churn（现参数 1.5% vs 下界 0.7%，通过） |
| C2 | 单笔净利检查按 **2(f+s)** 计 | min_net_profit 必须覆盖"这一笔 + 未来平掉这一笔"的两程成本（§21.7） |
| C3 | a_pt < θ_ext ≤ θ_t | 三带嵌套：可建偏斜带 / 停止加偏斜带 / 趋势带（`skip_if_price_extended` 由此接线） |
| C4 | θ_out < θ_t | 趋势进出迟滞，抑制 whipsaw |
| C5 | N·δ_unit·P·(θ_t − a_pt) ≤ β·D | 满档逆势偏斜的最坏止损 ≤ 回撤预算的 β 份额 |
| C6 | (q_b − q_floor)·P·p_t ≤ γ·D | 趋势敞口遭反转、回撤 p_t 才平的最坏损失 ≤ γ 份额（β + γ < 1） |
| C7 | Σ加亏损腿名义 ≤ ρ_f · Σ已实现收割利润 | **自融资不变量**：摊低成本的预算只能来自已收割利润，杜绝马丁格尔式放大 |
| C8 | π̂ > 1 − (a_pt − c)/θ_t | 品种准入（§2），按 symbol 历史拟合 |
| C9 | φ̄·\|Δ\|·P·E[持有期] ≤ 单次回归收益的小份额 | funding 是偏斜持有期的方向性成本，计入触发与持有决策（现恒为 0，需接线） |

**Whipsaw 成本模型**：θ_t 附近每次假确认 ≈ 一次砍腿 + 重建的 churn ≈ 2ρ·q_b·P·(f+s) + 步距不利偏移；用 C4 迟滞与 T_t 持续性把假确认频率压进预算。

## 7. 风控 = 硬不变量 + 吸收态（修复 §21.3 / §21.4）

风控从"事后生成风险事件"改为"动作发出前的不变量校验"（工程上即 ARCHITECTURE §4.3 的 RiskGuard 链）：

| 不变量 | 击穿处置 |
|---|---|
| gross = (L+S)·P ≤ 预算 × max_gross_exposure_ratio | 拒绝一切加仓动作。**ONLY_REDUCE 的精确定义：只放行降 \|Δ\| 或降 gross 的动作** |
| symbol 权益 ≥ B − D | → STOPPED：**拆对冲全平**（Δ→0 且 gross→0），吸收态，人工复核才能重启（终极止损出口） |
| 组合权益回撤 ≤ max_total_drawdown_pct | 全局 STOP（该参数由此接线） |
| C7 自融资账本余额 ≥ 0 | 拒绝加亏损腿动作 |
| 行情 / 持仓快照新鲜度未超时 | → PAUSED，不基于陈旧状态出动作 |

## 9. 验证规范（先于实现的验收标准）

**仿真场景**（合成价格路径，断言方向性结论）：

| 场景 | 路径 | 断言 |
|---|---|---|
| S1 纯震荡 | 正弦，振幅 ∈ (a_pt, θ_t) | PnL > 0 且随周期数增长；0 次趋势确认 |
| S2 单边趋势 | 单调走到 2θ_t | 亏损 ≤ C5+C6 预算；亏损腿到地板；**不锁死** |
| S3 V 型反转 | 单调至 θ_t 后全回归 | 趋势→结束→重锚→重建全链路走通；计数器清零；净损小（现状 bug 的回归测试） |
| S4 阈值横跳 | 在 θ_t ± ε 震荡 | 假确认次数 ≤ 迟滞预算；churn 成本 ≤ 上界 |
| S5 跳空 | 单 tick 穿越 θ_t | guard 拦住加仓，只执行减仓 |
| S6 横盘 | 平价 | 0 次交易（触发下界尊重成本） |
| S7 阴跌 | 缓慢漂移 + 高噪声 | 持续性/斜率确认不与 S2 同节奏误判 |

**性质测试**（任意路径恒成立）：C5/C6/C7 账本、gross 上限、STOPPED 吸收性、REANCHORING 后计数器归零、事件方向与 Δ 变化的一致性。

**品种准入**：按 §2 估 π̂ 与置信区间，低于 C8 下界的 symbol 不启用。

## 10. 提高最终盈利概率的设计升级

§1–§9 基线保证"可控、可验证"；本节回答"怎么把期望与胜率做大"。按杠杆从大到小：

### 10.1 盈亏几何反转（第一优先，纯参数层）

现默认参数 a=1.5% / θ=4% 的单注风报比 = a : (θ−a) = 1 : 1.67（倒挂），故需 π > 66%。
把几何反转为 a > θ−a（例：a=2%，θ=3%，maker 成本 c≈0.06%）：

```
π_req = 1 − (2 − 0.06)/3 ≈ 35%
```

vol 归一化下即 k₂/k₁ 从 ~2.7 降到 ~1.5。代价：θ 更紧 → 假趋势止损更频繁，π 本身随 θ 收紧而下降。
因此 π(a,θ) 曲面必须回测拟合，目标 max E(a,θ)·freq(a,θ) s.t. C5/C6——**几何是第一个该扫描的参数面，不是拍脑袋定的常数**。

### 10.2 市况过滤 regime gate（最大结构性杠杆）

策略的全部风险 = 市况误判，则直接测量市况：SKEW 建仓前置一个 gate，只在"回归证据存在"时开新偏斜。

- 统计量候选：窗口收益自相关（负 ⇒ 回归）、方差比 VR<1、Hurst H<0.5、窗口回归斜率 t-stat 低。
- 数学效果：E 对 π 线性。示例：无条件 π=60% ⇒ E=−0.24%/注（亏）；过滤后区间时段 π=75% ⇒ E=+0.36%/注（转正）。
- 工程落点：gate 是 Δ* 的前置乘子，regime ∈ {range, trend, unknown}，unknown 不开新偏斜（已有偏斜照常管理）。

### 10.3 趋势段回捕：止损分支从纯亏变部分对冲

基线趋势段只砍亏损腿到地板。升级：对地板化后的顺势敞口 Δ_t = q_b − q_floor 加**跟踪止盈**（回撤 p_t 触发），趋势延伸 e 时回捕 ≈ Δ_t·P·(e − p_t)。
效果：E 的亏损支变为 (θ − a − g)，π_req 进一步下降；收益剖面从"区间赚 / 趋势亏"变为"区间赚 / 趋势小赚小亏"——均值回归 + 趋势跟随的经典互补，跨市况更平衡，最终盈利概率上升。

### 10.4 执行成本：taker → maker

网格类触发天然适合挂限价单：c 从 0.14% → ~0.05%，π_req 直接下移。注意 maker 的成交不确定与逆向选择；plan→live 时采用"限价 + 超时转市价"的执行策略。

### 10.5 尺寸精细化

- 波动率目标化：δ_unit ∝ 1/σ̂，每注风险恒定；
- OU 拟合（可选）：估回归速度 κ，偏斜尺寸 ∝ κ×偏离——回归越快越可加；
- 分批止盈：偏斜回归 50% / 100% 分两批平，胜率升、单注方差降。

### 10.6 组合层

单 symbol E>0 后，N 个近独立 symbol 的组合盈利概率 ≈ Φ(√N·μ/σ)，随 √N 上升；准入仍按 C8 的 π̂ 门槛。
警惕：加密品种暴跌时相关性趋 1——10.2 的 regime gate 兼任组合级断路器（多数 symbol 同时进 TREND ⇒ 全局降杠杆）。

### 10.7 反模式警告

**禁止**用加深加档 / 放松 C7 来抬胜率：那是把概率从尾部借来——小赢频率上升、左尾变肥，长期盈利概率反而下降。C5 / C6 / C7 不可谈判。

### 优先级

1. 10.1 几何扫描（影响最大、零工程量）
2. 10.4 maker 成本（近乎免费）
3. 10.2 regime gate（最大结构性 edge，需新增统计模块）
4. 10.3 趋势回捕（状态机 TREND 段动作扩展）
5. 10.5 / 10.6（精细化与组合层）
