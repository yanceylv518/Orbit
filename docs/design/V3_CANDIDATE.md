## 2. 唯一预注册候选

候选名：`first_rung_loss_side_add_only`。

配置门：`events.profit_transfer.sizing.first_rung_loss_side_add_only`，默认 `false`。

启用时：

1. 利润搬运触发、盈利腿减仓、净利润门、成本门和自融资预算全部保持不变；
2. `target_step_count == 1` 时仍允许按现有预算补亏损腿；
3. `target_step_count > 1` 时只减盈利腿，`ADD_LOSS_SIDE` 数量固定为零；
4. 不改变目标净敞口、锚点、`a_pt`、`theta_t`、Regime Gate、趋势确认或趋势减仓；
5. 关闭时必须完全保持当前所有档位均可补亏损腿的行为。

该候选把补仓资格绑定到 excursion 深度：第一档仍处于离锚点最近、剩余回归空间最大的区域；进入后续档位后，不再为回归期限不确定的深度偏离追加逆势风险。它是结构开关，不扫描比例、阈值或市场专属参数。

## 3. 实现边界

- `actions.py` 只根据 `TargetExposureDecision.step_count` 决定是否生成 `ADD_LOSS_SIDE`，盈利腿减仓路径不变。
- 样例配置默认关闭；旧配置缺少该键时按关闭处理。
- 回放工具增加显式 variant `first_rung_loss_side_add_only`，不改变默认 `full` 口径。
- 不改 live 默认开关，不改变 RiskGuard、Funding、订单执行或账户同步。

## 4. 隔离验证

实现与单元测试完成后，只使用 `ADMISSION.md` 冻结的 20 个外层验证折，并运行 `fixed_ohlc`、`fixed_olhc`、`myopic` 三条 OHLC+Funding 路径。

GO 条件不变：每条路径均须净收益为正、至少 3/4 盈利市场、至少 11/20 盈利折、Funding 完整、最差折回撤不超过 5%，且 C8 统计门通过。任何一项失败即为 NO-GO；不根据验证结果修改本候选。

## 5. 实现结果

实现日期：2026-07-13。

- `actions.py` 仅在候选开启且 `target_step_count > 1` 时把补亏损腿预算置零，盈利腿减仓、利润门和成本门保持原样。
- action sizing 暴露 `loss_side_add_eligible`，便于计划和审计确认该档是否允许补仓。
- 回放新增显式 variant `first_rung_loss_side_add_only`；样例配置和缺省配置均保持关闭。
- 单元测试锁定第一档双动作、第二档只减盈利腿及回放配置隔离。

**V3 候选 FAIL，系统继续 NO-GO。** 开关保持默认关闭，不进入 testnet、paper 或 live。

当前 20 个验证折已经用于 V1、V2、V3 判定，不得继续作为下一候选的全新外样本。下一步应先获取时间上更新且未参与任何候选选择的 OHLC 与 Funding 数据，建立新的锁箱区间；在此之前可以继续做训练窗归因和候选设计，但不能宣称新的 go/no-go 结论。
