## 2. 唯一预注册候选

候选名：`neutralize_counter_trend_skew`。

配置门：`events.loss_side_reduction.sizing.neutralize_counter_trend_skew_only`，默认 `false`。

启用时：

1. 仍使用现有 `theta_t`、趋势进入持续确认和可选速度门，不改变何时确认趋势；
2. 上涨趋势只在当前净敞口为负时减少 SHORT，下降趋势只在当前净敞口为正时减少 LONG；
3. 目标净敞口固定为 0，单次动作完整消除当前反趋势偏斜；
4. 不跨过零点建立顺势敞口；达到中性后，同一趋势内后续价格阶梯不再产生趋势减仓动作；
5. 当前已中性或已顺势时不产生动作。

关闭时必须完全保持当前“翻向趋势方向并按比例阶梯减仓至地板”的行为。

## 3. 实现边界

- 目标几何在 `domain/strategy/exposure.py` 表达，动作 sizing 在 `actions.py` 只执行目标差额，避免计划目标与实际动作不一致。
- 不改变 Regime Gate、趋势确认、恢复、利润搬运、RiskGuard、Funding 或 live 默认开关。
- 回放工具只增加显式选择该候选的 variant，不改变默认 `full` 口径。

## 4. 隔离验证

实现和单元测试完成后，候选只按 `ADMISSION.md` 的 20 个验证折判定，并同时运行 `fixed_ohlc`、`fixed_olhc`、`myopic` 三条 OHLC+Funding 路径。

GO 条件不变：每条路径均须净收益为正、至少 3/4 盈利市场、至少 11/20 盈利折、Funding 完整、最差折回撤不超过 5%；统计 C8 门也必须通过。任何一项失败即为 NO-GO，不根据验证结果修改本候选。
