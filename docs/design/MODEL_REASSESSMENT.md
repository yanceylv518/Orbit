# Dynamic Dual Grid 模型重评

日期：2026-07-13  
结论：当前收益假设不成立，策略实现进入研究暂停

## 1. 重评原因

V1-V4 已依次验证参数/门控、趋势中和、逆势补仓深度和联合有界周期。BTC/ETH 与独立 BNB/SOL 锁箱均未通过统一准入门，且事件归因反复显示：利润搬运的局部正贡献会被趋势风险解除和旧仓恢复侵蚀。

同一合约的 Hedge Mode 双腿满足：

```text
delta = long_qty - short_qty
dPnL = delta * dPrice - costs - funding
```

因此双腿本身不产生收益；策略唯一可能的 alpha 是“偏离锚点后建立逆势净敞口，价格随后回归”。模型重评不再讨论如何搬运腿级利润，而直接检验该回归押注在固定持有期限和真实成本后的期望。

## 2. 期限诊断方法

新增纯研究估计器 `horizon_reversion_report` 和 CLI `backend/tools/analyze_reversion_horizon.py`：

1. 当前锚点偏离达到 `a=1.5%` 时，按反方向建立一单位归一化押注；
2. 价格先回到锚点带 `0.25%` 内，记为 reversion；先延伸到 `theta=4%`，记为 extension；
3. 两者均未发生且达到固定期限，按当时价格强制盯市退出，记为 timeout；
4. 每笔净收益扣除当前保守往返成本 `0.14%`；
5. 每次退出后在当前价格重锚，事件不重叠。

数据使用已打开、已不再具有新锁箱资格的 BTCUSDT、ETHUSDT、BNBUSDT、SOLUSDT：

- 15m：期限 4/16/32/96 ticks，对应 1h/4h/8h/24h；
- 1h：期限 1/4/8/24 ticks，对应 1h/4h/8h/24h；
- 共 `4 币种 x 2 周期 x 4 期限 = 32` 个诊断组合。

## 3. 结果

| 指标 | 结果 |
|---|---:|
| 成本后正期望组合 | 0/32 |
| 毛期望为正组合 | 22/32 |
| 最佳成本后期望 | -0.046136%/次 |
| 最佳组合 | SOLUSDT 1h，24h 期限 |
| 最佳毛期望 | +0.093864%/次 |
| 保守往返成本 | 0.140000%/次 |
| 各期限报告累计事件观测 | 12312 |
| reversion / extension / timeout | 2382 / 2074 / 7856 |

所有 32 个组合在成本后均为负。22 个组合存在微弱毛回归优势，但最好的毛期望 `+0.093864%` 仍低于 `0.14%` 往返成本；剩余组合连成本前期望也为负。累计观测中约 63.8% 以期限超时结束，说明偏离事件通常无法在目标期限内形成足以支付交易成本的回归。

## 4. 模型决策

**当前 Dynamic Dual Grid 收益假设不成立。** 失败并非主要来自某个减仓比例、档位或退出 tick，而是无条件锚点回归信号没有稳定的成本后正期望。

从即日起：

- 策略保持 `plan_only/read_only`，不得进入 testnet、paper 或 live；
- 停止继续增加 V5 式仓位几何和恢复旧亏损腿逻辑；
- 已有账户同步、快照、计划、风控审计和回放基础设施保留，它们是模型无关的平台能力；
- 当前策略实例应视为研究样例，而不是待上线策略。

## 5. 恢复策略开发的硬条件

新的策略模型必须先定义可独立测量的 alpha 信号，再设计仓位状态机。至少满足：

1. 信号不得只由“价格离锚点达到固定百分比”构成，必须有预注册的条件信息或市场状态；
2. 在训练数据上逐市场计算真实盯市收益，扣除手续费、双边滑点和 Funding；
3. 至少 30 个非重叠事件，成本后单次期望为正，且置信下界高于零；
4. 参数、市场、期限和退出规则必须在新锁箱打开前冻结；
5. 新模型不得依赖恢复旧亏损腿来制造腿级已实现利润；任何仓位迁移都必须用净敞口和总权益解释；
6. 只有通过全新未见数据后，才允许恢复 paper/testnet 工程。

下一研究任务不是实现交易动作，而是建立 alpha 候选协议：明确候选特征、标签、非重叠采样、成本模型、训练/锁箱隔离和废弃标准。
