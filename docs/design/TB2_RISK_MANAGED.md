## 3. 冻结风险 overlay 网格

TB2 不包含 TB1 的 `10% + 无节流` 基线候选；该配置只作为同窗口诊断基准。候选固定为：

| ID | 目标组合波动率 | 回撤阈值 | 阈值后风险倍率 | 类型 |
|---|---:|---:|---:|---|
| `vol06_none` | 6% | 无 | 1.0 | 纯规模控制 |
| `vol08_none` | 8% | 无 | 1.0 | 纯规模控制 |
| `vol10_dd10` | 10% | 10% | 0.5 | 纯回撤节流 |
| `vol10_dd15` | 10% | 15% | 0.5 | 纯回撤节流 |
| `vol06_dd10` | 6% | 10% | 0.5 | 组合 |
| `vol06_dd15` | 6% | 15% | 0.5 | 组合 |
| `vol08_dd10` | 8% | 10% | 0.5 | 组合 |
| `vol08_dd15` | 8% | 15% | 0.5 | 组合 |

回撤节流只在每周信号收盘时评估。先用旧仓结算截至该收盘的价格、Funding、换手和成本，更新窗口内权益与历史峰值，再计算：

```text
drawdown(t) = 1 - equity(t) / running_peak(t)
risk_scale(t) = 0.5 if drawdown(t) >= threshold else 1.0
overlay_weight_i(t) = base_target_weight_i(t) * risk_scale(t)
```

缩放后的目标权重仍受 gross cap `1.0` 约束，并在下一根 K 线执行。节流不在两次周频决策之间强制平仓，不使用未来价格，不增加止损、恢复等待期或滞回参数。回撤低于阈值后，下一次周频决策恢复 `risk_scale=1.0`。

## 4. 每步训练选择

每个 expanding training 段分别运行全部 8 个候选。候选进入训练池必须同时满足：

1. 年化净收益 `> 0`；
2. Sharpe `>= 0.50`；
3. 最大回撤 `<= 20%`；
4. 完整 365 天折中，盈利折数严格过半；
5. 原 TB1 数据质量与 Funding 完整性门通过。

训练池非空时，只按以下冻结顺序选择一个候选：最大回撤更低、Sharpe 更高、年化净收益更高、候选 ID 字典序更小。训练池为空时，该 walk-forward 步记为 `TRAINING_FAIL`，不得查看该步 OOS 来补选候选，TB2 最终直接 FAIL。

## 5. OOS 记账与准入门

每步只能把该步历史训练选出的唯一候选用于紧随其后的 OOS 窗口。输出每个窗口的总净收益、年化净收益、年化波动率、Sharpe、最大回撤、盈利年度折、换手、成本、Funding、节流触发次数和风险倍率轨迹。

两个 OOS 窗口的逐期净收益按时间顺序拼接形成聚合序列。**每一个 OOS 窗口和聚合序列**都必须同时满足：

1. 年化净收益 `> 0`；
2. Sharpe `>= 0.50`；
3. 最大回撤 `<= 20%`；
4. 盈利 365 天折数严格过半。

还必须满足两步训练均成功、数据质量门通过。任一条件失败即为 `TB2_FAIL`；全部通过才是 `TB2_PASS`。

## 6. 规模控制与形状改善的区分

每个 OOS 同时运行只读的 TB1 `10% + 无节流` 基线，用于报告差异，不参与选参。报告必须明确：

- 选中 `vol06_none` 或 `vol08_none` 时标记为 `SCALE_ONLY`；降低仓位压低回撤是合法结果，Sharpe 近似不变即可接受；
- 选中带回撤阈值的候选时标记为 `THROTTLED`，并记录相对基线的 Sharpe 变化与回撤下降百分点；
- 不得把纯规模缩小描述为改善了收益/回撤形状。
