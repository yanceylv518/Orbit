## 1. 研究定位

TB3 是趋势篮子的最后一道历史回测门。它不修改 TB1/TB2 已冻结的 alpha 方向，而是按用户真实风险偏好重新定义“可接受”：允许趋势策略存在亏损年份和较长恢复期，但要求成本后长期为正、每单位回撤获得足够收益、下行风险可控。

TB3 的历史约 `2021-2026` 已被 TB1/TB2 从其他角度检视，因此本轮 verdict 只能称为**回测确认**，不是全新锁箱证据。TB3 PASS 只表示完整系统具备进入 TB4 纸面前向测试的资格，不授权 testnet/live；真正未见样本只能来自未来时间。

## 2. 冻结系统

- 12 个 Binance USD-M perpetual 市场与原输入 SHA-256；

- 最终清仓计成本。

不使用 TB2 的回撤节流。唯一训练选择是组合目标波动率：

| ID | 目标组合波动率 |
|---|---:|
| `vol10` | 10% |

每个 expanding training 段只允许选择“成本后总净收益 `> 0` 且训练最大回撤 `<= 25%`”的候选，并从中冻结目标波动率最高者；若同波动率重复，按候选 ID 字典序。`25%` 是相对用户 `30%` 最大容忍度预留的 5 个百分点训练缓冲，不根据 OOS 结果改变。训练池为空时，该步记为 `TRAINING_FAIL`，不得打开对应 OOS。

## 4. 指标定义

所有指标基于逐期成本后净收益复利计算，无风险利率和最低可接受收益均按 0。

### 4.1 Calmar

```text
Calmar = annualized_net_return_pct / max_drawdown_pct
```

若最大回撤为 0 且年化净收益为正，视为正无穷并通过 Calmar 门；报告使用 `null` 表示该无穷值并单独记录通过状态，避免产生非标准 JSON `Infinity`。

### 4.2 Sortino

```text
downside_deviation = sqrt(mean(min(period_return, 0)^2))
Sortino = mean(period_return) / downside_deviation * sqrt(periods_per_year)
```

若下行偏差为 0 且平均收益为正，处理方式同 Calmar 的正无穷情形。

### 4.3 滚动 12 个月

- 每个滚动窗口固定为 `2190` 根 4h 收益（365 天）；
- 每 `6` 根 4h K 线，即每自然 24 小时取一个结束点；
- 窗口收益按内部逐期净收益复利；
- 输出最差滚动 12 个月收益、总窗口数、正收益窗口数和正收益占比；
- 收益严格大于 0 才算正窗口。

### 4.4 最大回撤持续时长

权益低于此前历史峰值的连续 4h 期数记为 underwater duration；创出或回到新高时归零。最大连续期数按：

```text
max_drawdown_duration_months = max_underwater_periods / periods_per_year * 12
```

窗口结束时尚未恢复的回撤也计入最大持续时长。

## 5. 冻结准入门

每一个已打开的 OOS 窗口和按时间拼接的完整 OOS 聚合序列都必须同时满足：

1. 成本后总净收益 `> 0`；
2. 最大回撤 `<= 30%`；
3. Calmar `>= 0.50`；
4. Sortino `>= 0.70`；
5. 最差滚动 12 个月收益 `>= -30%`；
6. 正收益滚动 12 个月窗口占比 `>= 55%`；
7. 最大回撤持续时长 `<= 18` 个月；
8. 12/12 固定市场的数据质量与 Funding 完整性门通过。

两步训练都必须产生唯一候选，两步 OOS 和聚合都必须通过。任一条件失败即为 `TB3_FAIL`；全部通过才是 `TB3_PASS`。报告同时列出最差净收益、Calmar、Sortino、滚动 12 个月收益/占比和回撤时长分别来自哪个 OOS。
